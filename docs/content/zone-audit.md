## Confidence Framework

Each data point in the scorecards carries a confidence tag:

| Tag | Meaning |
|-----|---------|
| **CONFIRMED** | Verified in DB + code + tested at runtime |
| **HIGH** | Present in DB or code and cross-referenced against a second source |
| **MEDIUM** | Inferred from cross-referencing multiple sources (DB + client files, etc.) |
| **LOW** | Based on client file names, directory structure, or single-source inference |
| **ASSUMPTION** | MMO industry norms or developer intent guess with no direct evidence |

Throughout this document, confidence tags are applied at the individual data-point level,
not at the zone level. A zone might have CONFIRMED client art but LOW mission data.

---

## Verdict Definitions

Each zone receives one of five verdicts reflecting its overall functional completeness
from the perspective of a player or developer attempting to use it:

| Verdict | Meaning |
|---------|---------|
| **PLAYABLE** | Has space script, NPCs, missions, navigation mesh, and client art. A player could load in and play through meaningful content. |
| **PARTIAL** | Has some functional content (transport, NPCs, partial missions) but incomplete gameplay. A player could enter but would hit dead ends. |
| **SHELL** | Has client art and a server space definition, but zero gameplay content. The zone loads but there is nothing to do. |
| **DATA-ONLY** | Has DB entries (world IDs, stargate records, mission labels) but no server space definition or functional script. Cannot be loaded as a playable zone. |
| **MISSING** | Exists only as a concept reference or placeholder name. No meaningful data in any source. |

A sub-classification of PARTIAL is noted where a zone's only functional content is
transport infrastructure (ring transporters, beam pads) with no missions or combat.

## Data Sources Overview

The audit cross-references the following sources:

### Server-Side Definitions

- **spaces.xml** — 24 spaces total (16 persistent, 8 instanced). This is the canonical
  list of zones the server can load.
- **cell_spaces.xml** — Cell-level spatial partitioning for persistent zones only.
  Instanced spaces do not appear here.
- **Python space scripts** — Per-zone `.py` files in `python/` that define entity spawns,
  mission triggers, ring transporters, region events, and NPC tags.
- **Space .script files** — Compiled or authored script-graph files with node counts,
  separate from Python scripts. Co-located with space data.

### Navigation

- **NavMesh files (.nav)** — Only 5 exist: `agnos.nav`, `castle_cellblock.nav`,
  `harset.nav`, `harset_storagerm.nav`, `sgc_w1.nav`. Generated by the NavBuilder tool
  from Recast/Detour. Required for AI pathfinding and valid player movement.

### Database

- **world_id** — Each zone has a numeric world identifier in the DB.
- **has_script** — DB boolean flag indicating whether the zone has an associated script.
  This flag is not always accurate (some zones with scripts show `false`).
- **Stargate entries** — DB records linking zones to stargate IDs for the gate network.
- **Spawnlist entries** — DB records defining NPC/entity spawn points per zone.
- **Mission labels** — DB mission records tagged with zone names.

### Client Assets

- **.umap files** — Unreal Engine map tiles. Each zone directory contains between 2 and
  892 .umap files. Total across all 24 zone directories: 4,618 files. The `Login_Map`
  directory exists in client data but has no server space definition (client-only).

| Aspect | Status | Details |

**Verdict: SHELL**

no script activates them. The 8 missions labeled "Agnos" in the DB have no implementation.
This zone was clearly planned as a major content area but development halted before any
gameplay was wired up. The navmesh suggests it was further along in the pipeline than most
shell zones — possibly the next zone that would have been brought online.

### Zone: Castle

| Aspect | Status | Details |
|--------|--------|---------|
| Client Art | 145 .umap tiles | **HIGH** — Medium-sized zone |

- Mission hooks for IDs 701 and 702
- Named regions: ThroneRoom, Courtyard, Infirmary
- Livewire minigame integration
- 76 .script graph nodes suggest additional authored logic

which means AI pathfinding would not work and player movement validation is absent. The
two scripted missions (701-702) are functional in the script but represent only a fraction
of what a full zone would need. The Livewire minigame is a notable feature — it shows this
zone had active gameplay development. Without navmesh, this zone is playable in a degraded
mode (players could load in, interact with NPCs, trigger missions) but AI-driven content
would fail.
