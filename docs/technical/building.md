# Building Cimmeria

## Automated (Recommended)

Run the bootstrap script to download, patch, and build everything:

```powershell
pwsh setup-dependencies.ps1
```

Then open `W-NG.sln` in Visual Studio and build `Debug|x64`.

See [../bootstrap/README.md](../bootstrap/README.md) for full details on the
bootstrap process.

## Manual Build Steps

If you prefer to build dependencies individually, or need to rebuild a single
library, use the standalone build scripts in `bootstrap/`. Each script finds
Visual Studio via `vswhere.exe` automatically ‚Äî you don't need to run from a
VS command prompt.

### Boost 1.55.0

```cmd
bootstrap\init-boost.bat      &:: Bootstrap b2 (one-time)
bootstrap\build-boost.bat     &:: Build static libraries for x64
```

**Requires:** `external/boost/user-config.jam` (generated by `setup-dependencies.ps1`,
or create manually from `bootstrap/templates/user-config.jam.template`).

**Output:** `external/boost/lib64/lib/libboost_*-vc145-mt-*.lib`

**Libraries built:** date_time, math, python, system, thread (plus dependencies
like chrono).

### OpenSSL 1.0.1e

```cmd
bootstrap\build-openssl.bat
```

**Requires:** Perl on PATH (Strawberry Perl or Git for Windows bundled Perl).

**Output:**
- `external/openssl/lib64/libeay32MT.lib` (release)
- `external/openssl/lib64/libeay32MTd.lib` (debug)

The script builds both debug and release configurations. It creates a safe Perl
wrapper directory to prevent Git's `usr/bin/link.exe` from shadowing MSVC's linker.

### SOCI 3.2.1

```cmd
bootstrap\build-soci.bat
```

**Output:**
- `lib64/debug/libsoci_core_3_2.lib`
- `lib64/debug/libsoci_postgresql_3_2.lib`
- `lib64/release/libsoci_core_3_2.lib`
- `lib64/release/libsoci_postgresql_3_2.lib`

**Requires:** `external/postgresql/include/` (PostgreSQL headers for the backend).

### Other Dependencies

These don't need separate build steps:

- **PostgreSQL** ‚Äî Pre-built binaries, just headers + `libpq.lib`
- **Python** ‚Äî Headers + import libs extracted from MSI
- **SDL** ‚Äî Pre-built development libraries
- **Recast/Detour** ‚Äî Built by `Recast.vcxproj` as part of the solution

## Solution Build

Build configurations available in `W-NG.sln`:

| Configuration | Description |
|---|---|
| `Debug\|x64` | Development build (recommended) |
| `Release\|x64` | Optimized production build |
| `UnoptRelease\|x64` | Release CRT, no optimization (debugging release issues) |
| `MinSizeRel\|x64` | Size-optimized build |

### From Visual Studio

1. Open `W-NG.sln`
2. Select `Debug` / `x64` from the toolbar dropdowns
3. Build > Build Solution (Ctrl+Shift+B)

### From Command Line

```cmd
msbuild W-NG.sln /p:Configuration=Debug /p:Platform=x64 /m
```

The `/m` flag enables parallel project builds.

## Compatibility Patches

The vendored dependencies (Boost 1.55, OpenSSL 1.0.1e, SOCI 3.2.1) predate
VS2015 and require source patches to compile with modern MSVC. These are stored
in `bootstrap/patches/` and applied automatically by `setup-dependencies.ps1`.

If you extract dependencies manually, you must apply these patches yourself:

```powershell
copy bootstrap\patches\boost_auto_link.hpp      external\boost\boost\config\auto_link.hpp
copy bootstrap\patches\soci_platform.h          external\soci\src\core\soci-platform.h
copy bootstrap\patches\soci_backends_config.h   external\soci\src\core\soci_backends_config.h
copy bootstrap\patches\openssl_e_os.h           external\openssl_src\e_os.h
copy bootstrap\patches\openssl_e_padlock.c      external\openssl_src\engines\e_padlock.c
```

See [../bootstrap/patches/README.md](../bootstrap/patches/README.md) for details
on what each patch fixes.
